## Phase 3: Design `/train` Loop Architecture âś“
**Goal:** Design the autonomous training loop protocol with clear convergence criteria and regression safety.

1. [x] [RSH] **Define loop protocol:**
   - Created `.workflow/commands/train.md` with full loop protocol
   - 11-step epoch cycle: simulate â†’ evaluate â†’ check termination â†’ select target â†’ restore point â†’ diagnose â†’ implement â†’ mini-verify â†’ full re-evaluate â†’ check regressions â†’ log
   - Fix selection: priority (P0>P1>P2), then normalized delta, skip content-signal systems
   - One system per epoch (isolate variables)

2. [x] [RSH] **Define convergence and stall criteria:**
   - Converged: all PASS/WARN, no FAIL remaining
   - Content-converged: all remaining FAILs have signal_source: "content"
   - Stalled: 2 epochs with same PASS/FAIL counts and no metric movement >5%
   - Regression halt: PASSâ†’FAIL on non-target, 2 alternatives failed â†’ skip
   - Max epochs: configurable via `--epochs N` (default 5)
   - Context budget: checkpoint and instruct `/train --continue`

3. [x] [RSH] **Define regression safety protocol:**
   - Before fix: `git stash push -m "train-epoch-N-before-<system>"`
   - On regression: `git stash pop`, log failed approach with description and result
   - Max 2 alternatives per system per training run
   - Failed approaches tracked in session state to avoid retrying
   - Multi-system regression: higher-priority system wins, lower-priority fix is skipped

4. [x] [DOC] **Document the architecture:**
   - Full skill file at `.workflow/commands/train.md` (261 lines)
   - Includes: ASCII loop flowchart, convergence table, regression protocol, training state schema
