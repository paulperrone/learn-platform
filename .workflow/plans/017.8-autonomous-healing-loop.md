## Summary

Split the healing system into two complementary tools: `/heal` (upgraded diagnostic — single iteration, human-guided) and `/train` (autonomous training loop — multi-epoch gradient descent on the engine). `/train` runs full simulations (30 sessions × 10 profiles), evaluates against learning-science targets, diagnoses the worst failure, implements a fix, re-simulates to verify, checks for regressions, and either loops again or stops when converged/stalled.

**Depends on:** Plan 017.7 ✅ (healing loop infrastructure, targets, evaluation engine)

**Motivation:** The healing system infrastructure (targets, evaluation, epoch tracking) exists but the loop is manual — a human must trigger `/heal`, wait, review, trigger again. The engine has 4 failing systems (interleaving, mastery convergence, FIRe compression, review/new balance) that need iterative optimization. An autonomous loop that runs heavy simulations, diagnoses, fixes, and re-tests is the "neural network training" analogy: simulations = forward pass, targets = loss function, diagnosis = backprop, code changes = weight updates.

**Known state entering this plan:**
- FIRe compression: 0% → 8.5% (Phase 7 fix, needs further iteration to reach 20%)
- Interleaving: PASS → FAIL (0.071 → 0.144, warmup diversity regression from reduced materialization)
- Mastery convergence: WARN → FAIL (3 → 1, fewer initial mastered topics)
- Review/new balance: FAIL (0.88, target 0.50-0.70)
- Simulation metrics don't account for implicit mastery (reporting bug)

## Phase 3: Design `/train` Loop Architecture
**Goal:** Design the autonomous training loop protocol with clear convergence criteria and regression safety.

1. [ ] [RSH] **Define loop protocol:**
   - Max epochs per `/train` invocation (default: 5, configurable via `--epochs N`)
   - Each epoch: full `just heal-epoch 30 42` → evaluate → diagnose worst FAIL → implement fix → `just test` → mini-verify → full re-evaluate
   - Between epochs: compare against previous epoch. If regression detected, rollback and try alternative approach.
   - Fix selection: highest priority (P0 > P1 > P2), then largest delta from target

2. [ ] [RSH] **Define convergence and stall criteria:**
   - **Converged:** All systems PASS (or all FAILs are content-signal-source, not engine)
   - **Stalled:** 2 consecutive epochs with no system improving (same PASS/FAIL counts, no metric movement > 5%)
   - **Regression halt:** Any system moves from PASS → FAIL that wasn't the fix target. Rollback fix, try alternative. If 2 alternatives fail, skip system and move to next.
   - **Context budget:** If approaching context window limit, checkpoint progress and instruct user to re-run `/train` to continue

3. [ ] [RSH] **Define regression safety protocol:**
   - Before each fix: `git stash` or create a restore point
   - After fix + mini-verify: run full epoch to check ALL systems
   - If regression: `git stash pop` to restore, log the failed approach, try alternative
   - Track failed approaches per system to avoid retrying the same fix

4. [ ] [DOC] **Document the architecture:**
   - Write architecture section in the `/train` skill file
   - Include: loop flowchart (text), convergence table, regression protocol, output format
   - Include: epoch report template showing before/after/delta per system

**Validation:** Architecture document is complete, reviewed, and covers all edge cases (convergence, stall, regression, context limits). Protocol is clear enough that the skill can follow it autonomously.

## Phase 4: Implement `/train` Skill
**Goal:** Build the `/train` command that runs the full autonomous training loop.

1. [ ] [IMP] **Create `.claude/commands/train.md` skill file:**
   - YAML frontmatter with description, allowed-tools
   - Arguments: `--epochs N` (default 5), `--target <system>` (focus on one system), `--dry-run` (evaluate only, no fixes), `--continue` (resume from previous `/train` state)
   - Full loop protocol from Phase 3 architecture

2. [ ] [IMP] **Implement epoch orchestration:**
   - Step 1: Run `just heal-epoch 30 42` (full simulation + evaluation)
   - Step 2: Parse evaluation.json, identify highest-priority FAIL
   - Step 3: Read system playbook from `/heal`, trace root cause through code
   - Step 4: Design and implement fix (following `/heal` Step 3-4 pattern)
   - Step 5: Run `just test` — must pass
   - Step 6: Run `just heal-verify <system>` — mini-sim verification
   - Step 7: Run full `just heal-epoch 30 42` — check all systems
   - Step 8: Compare epoch N vs epoch N-1. Log deltas. Check for regressions.

3. [ ] [IMP] **Implement convergence and stall detection:**
   - After each epoch: check convergence criteria from Phase 3
   - If converged: output final report, update baselines, stop
   - If stalled: output stall report with what was tried, suggest human review areas, stop
   - If max epochs reached: output progress report, note remaining failures, stop

4. [ ] [IMP] **Implement regression rollback:**
   - Before each fix: `git stash push -m "train-epoch-N-restore-point"`
   - If regression detected: `git stash pop`, log failed approach
   - Track failed approaches in session state (avoid retrying)
   - If 2 approaches fail for same system: skip, move to next system

5. [ ] [IMP] **Implement progress reporting:**
   - After each epoch: print status table (system, before, after, delta, status)
   - After loop completion: print summary (epochs run, systems fixed, systems remaining, convergence status)
   - Write structured output to `simulations/reports/training/` for history

6. [ ] [TST] **Unit test the skill logic:**
   - Test convergence detection with mock evaluation data
   - Test stall detection (2 epochs, no improvement)
   - Test regression detection (PASS → FAIL)
   - Verify rollback protocol works with git stash

**Validation:** `/train --dry-run` runs evaluation and reports status without making changes. `/train --epochs 1` runs a single epoch end-to-end (simulate → diagnose → fix → verify → report).
