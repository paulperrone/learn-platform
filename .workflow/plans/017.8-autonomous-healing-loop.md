## Summary

Split the healing system into two complementary tools: `/heal` (upgraded diagnostic — single iteration, human-guided) and `/train` (autonomous training loop — multi-epoch gradient descent on the engine). `/train` runs full simulations (30 sessions × 10 profiles), evaluates against learning-science targets, diagnoses the worst failure, implements a fix, re-simulates to verify, checks for regressions, and either loops again or stops when converged/stalled.

**Depends on:** Plan 017.7 ✅ (healing loop infrastructure, targets, evaluation engine)

**Motivation:** The healing system infrastructure (targets, evaluation, epoch tracking) exists but the loop is manual — a human must trigger `/heal`, wait, review, trigger again. The engine has 4 failing systems (interleaving, mastery convergence, FIRe compression, review/new balance) that need iterative optimization. An autonomous loop that runs heavy simulations, diagnoses, fixes, and re-tests is the "neural network training" analogy: simulations = forward pass, targets = loss function, diagnosis = backprop, code changes = weight updates.

**Known state entering this plan:**
- FIRe compression: 0% → 8.5% (Phase 7 fix, needs further iteration to reach 20%)
- Interleaving: PASS → FAIL (0.071 → 0.144, warmup diversity regression from reduced materialization)
- Mastery convergence: WARN → FAIL (3 → 1, fewer initial mastered topics)
- Review/new balance: FAIL (0.88, target 0.50-0.70)
- Simulation metrics don't account for implicit mastery (reporting bug)

## Phase 3: Design `/train` Loop Architecture ✓
**Goal:** Design the autonomous training loop protocol with clear convergence criteria and regression safety.

1. [x] [RSH] **Define loop protocol:**
   - Created `.workflow/commands/train.md` with full loop protocol
   - 11-step epoch cycle: simulate → evaluate → check termination → select target → restore point → diagnose → implement → mini-verify → full re-evaluate → check regressions → log
   - Fix selection: priority (P0>P1>P2), then normalized delta, skip content-signal systems
   - One system per epoch (isolate variables)

2. [x] [RSH] **Define convergence and stall criteria:**
   - Converged: all PASS/WARN, no FAIL remaining
   - Content-converged: all remaining FAILs have signal_source: "content"
   - Stalled: 2 epochs with same PASS/FAIL counts and no metric movement >5%
   - Regression halt: PASS→FAIL on non-target, 2 alternatives failed → skip
   - Max epochs: configurable via `--epochs N` (default 5)
   - Context budget: checkpoint and instruct `/train --continue`

3. [x] [RSH] **Define regression safety protocol:**
   - Before fix: `git stash push -m "train-epoch-N-before-<system>"`
   - On regression: `git stash pop`, log failed approach with description and result
   - Max 2 alternatives per system per training run
   - Failed approaches tracked in session state to avoid retrying
   - Multi-system regression: higher-priority system wins, lower-priority fix is skipped

4. [x] [DOC] **Document the architecture:**
   - Full skill file at `.workflow/commands/train.md` (261 lines)
   - Includes: ASCII loop flowchart, convergence table, regression protocol, training state schema
   - Includes: epoch status table template, completion report template
   - Includes: dry-run mode, continue mode, key constraints

**Validation:** Architecture document complete. Covers convergence, stall, regression, context limits, dry-run, and continue. Protocol is clear enough for autonomous execution in Phase 4.
